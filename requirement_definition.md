コーディングテスト機能 要件定義（MVP）

1. 目的

本機能は、以下を目的とする。

アルゴリズム系コーディングテストを解ける環境を提供する

将来の「業務再現課題」と 同一基盤 で動作させる

自分の実装力・思考速度・ミス傾向を可視化する

※ 面接特化ではなく、実務力向上を主目的とする。

2. 想定ユーザー

ユーザー：本人のみ（認証不要）

利用シーン：

ウォームアップ

アルゴリズム力の維持

実務課題前の準備運動

3. スコープ
   3.1 対象とするもの

Python によるコーディングテスト

単一ファイルの実装問題

自動テストによる正誤判定

3.2 対象外（MVP ではやらない）

他言語対応

ランキング

解説生成

AI ヒント

インフラ依存問題

4. 機能要件
   4.1 問題管理

問題は ファイルベース で管理する

各問題は以下の情報を持つ

id: ct-001
title: 配列内の 2 数の和
difficulty: easy | medium | hard
category:

- array
- hash
  time_limit_sec: 1
  memory_limit_mb: 256

4.2 問題本文

Markdown 形式

以下を必須とする

問題概要

入力仕様

出力仕様

制約条件

サンプル入力・出力

4.3 実装形式

提出コードは 単一 Python ファイル

関数シグネチャは問題側で固定する

例：

def solve(nums: list[int], target: int) -> bool:
pass

4.4 実行・判定

サーバー側で pytest を実行

非公開テストケースを含める

すべてのテスト通過をもって「正解」とする

4.5 実行制限

実行時間制限を設ける

メモリ制限は MVP では 論理制限のみ（厳密管理しない）

5. 非機能要件
   5.1 安定性

テスト実行失敗時はクラッシュせず、エラーメッセージを返す

5.2 セキュリティ（最低限）

実行はローカル or 隔離環境

ファイル IO は禁止

ネットワークアクセスは禁止

※ 完全なサンドボックスは MVP 対象外（正確性 60%）

6. UI 要件（最低限）
   6.1 問題一覧

ID

タイトル

難易度

カテゴリ

6.2 問題詳細画面

問題文表示

コードエディタ

実行ボタン

実行結果表示（成功 / 失敗 / エラー）

7. データ保存（自分用）

実行履歴を保存

問題 ID

実行日時

成否

実行時間（参考値）

8. 将来拡張を見据えた設計方針

課題タイプは mode: coding_test として定義

業務再現課題と共通インターフェースを持つ

Runner 層を抽象化し、将来 Docker 実行に切り替え可能にする

9. 既知の不確実点

以下は MVP 時点では厳密に定義しない（正確性 50%）。

実行時間の厳密測定方法

メモリ使用量の測定

悪意あるコードへの完全対策

10. 完了条件（MVP）

1 問以上のコーディングテストが解ける

正誤が自動判定される

業務再現課題と同じ土台で動く
